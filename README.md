# saju_youth — 사주 오행 팔찌 공방 프로젝트

"사주 오행 팔찌 공방"을 후킹(미끼)으로 사용하는 마케팅/전도 퍼널 프로젝트.

## 🌐 배포 주소 (GitHub Pages)
- 진단 페이지(QR용): **https://capshinsmg-blip.github.io/saju_youth/**
- 관리자: **https://capshinsmg-blip.github.io/saju_youth/admin.html**
- `main` 브랜치에 push하면 약 1분 뒤 자동 반영됩니다.

## 퍼널 한눈에 보기

```
[온라인] 인스타 메타광고 ─┐
                          ├─→ 연락처·사주정보 수집 → 공방 예약 → 방문 → 상담 연계(최종 KPI)
[오프라인] 럭키드로우 부스 ─┘
```

방문 진행: 웰컴(차) → 사주 풀이 20분 → 팔찌 제작 30~40분 → 패키징/포토존

## 핵심 수치
- 판매가: **7,000원** / 원가: **5,000원 이내**
- 진행자("잎사귀") 1명 주관 — 양성 필요
- 장소: 맬로우 스페이스 (대여)

## 🔮 3초 사주 진단 웹페이지 (`index.html`)
QR 접속용 모바일 랜딩. 검증된 만세력 라이브러리([lunar-javascript](https://github.com/6tail/lunar-javascript), 입춘·24절기 정확 적용)로 사주 8자·오행·십신을 계산해 **금전운/연애운 %**(왜 이 수치인지·상태 등급 해석 포함)와 **천간지지·일간 설명**, **부족 오행 → 추천 색/원석/방위**를 보여주고, 🔒 심화 리포트는 공방 방문(상담) 유도.

화면 흐름: 랜딩 → 정보입력(**양력/음력·윤달 지원**) → 결과(1차 일부 잠금) → **해설지**(인스타 팔로우 후 전환, 성향·오행·금전/연애 공개) → **2차 잠금**(올해 월운+10년 대운/세운/조심시기는 예약 시 공개) → **신청 폼**(연락처+고민+개인정보/광고 수신 **동의**) → 완료.

> 차별화: 팔로워용 **해설지**는 운세 상세가 2차 잠금(예약 유도), **관리자(admin.html)**는 상담사용으로 대운·세운·월운까지 전체 공개.

- **수정할 값**: `index.html` 상단 `CONFIG` — 인스타 핸들/URL, 예약폼 링크, 브랜드명 (현재 TODO 더미값)
- **로컬 미리보기**: `node preview_server.js` → http://localhost:8123
- **배포**: 정적 파일이라 **GitHub Pages**(Settings→Pages→main/root) 또는 Vercel/Netlify에 그대로 올리면 됨
- **계산 데이터는 기기에서만 처리** (서버 전송·저장 없음 → 개인정보 부담 적음)

### 구성 파일
- `index.html` — 공개 진단 퍼널 (랜딩·입력·결과·해설지·신청·완료)
- `saju.js` — 사주 계산 + 해석 공용 엔진 (index·admin 공용)
- `admin.html` — **관리자 신청자 풀이 조회**. 로그인 후 신청자별 세부 만세력 + 상담 작성란
- `db.js` + `supabase-config.js` — 신청 저장/조회 레이어. **Supabase 연동 시 클라우드 저장**, 미설정 시 localStorage 폴백
- `docs/04_supabase_연동_가이드.md` — Supabase 프로젝트 생성·테이블·RLS·관리자 계정·키 설정 5단계
- `docs/05_예약_일정_마이그레이션.sql` — 예약 날짜/중복차단/담당자/취소·보관함 기능용 컬럼·인덱스·RPC·staff 테이블 (SQL Editor에 실행)

### 백엔드(Supabase) 동작
- **미설정 상태**(기본): 신청은 이 브라우저 localStorage에만 저장, admin은 임시 비번(`saju2026`)+샘플로 검토 가능
- **설정 후**: 신청 → 클라우드 DB 저장, admin은 등록된 관리자 이메일/비번 로그인 → 어디서나 조회. RLS로 공개는 INSERT만·조회는 관리자만.

### 인스타 팔로우 → 해설지 (동작 방식)
웹은 팔로우 여부를 **자동 검증할 수 없음**(메타 API 제한). 현재는 버튼 클릭 시 인스타를 새 탭으로 열고 → 해설지 화면으로 전환하는 **명예제** 방식. 실제 리드 확보는 **ManyChat 등 DM 자동응답**(키워드 DM → 해설지 발송) 병행 권장.
